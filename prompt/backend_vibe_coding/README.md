# Backend Vibe Coding - 后端 AI 编程助手

> 基于多种 AI 编程方法论的后端开发提示词库

---

## 项目概述

本项目是一套结构化的 AI 编程提示词库，专为后端 API 开发设计。它整合了 **VibeCoding**、**GSD**、**SpecCoding** 三种方法论的优点，提供从项目初始化到代码生成的完整工作流。

## v6.0 使用流程

### 1. 从入口开始

所有任务都从 `v6_refactored/A_Decision.txt` 开始：

1. 复制 `prompts/v6_refactored/A_Decision.txt` 的内容给 AI
2. AI 会引导你判断：
   - **任务类型**：Bug修复 / 配置变更 / 小优化 / 文档更新 / 新功能开发
   - **项目规模**：小项目 / 大项目
3. 根据判断结果，AI 会告诉你使用哪个提示词文件

### 2. D 快速通道（4种场景）

| 场景 | 提示词文件 | 适用情况 |
|------|-----------|---------|
| Bug 修复 | `D_quick/D1_bugfix.txt` | 修复已知问题 |
| 配置变更 | `D_quick/D2_config.txt` | 修改配置文件 |
| 小优化 | `D_quick/D3_optimize.txt` | 性能优化 |
| 文档更新 | `D_quick/D4_doc.txt` | 更新文档 |

### 3. S_simple 小项目流程（14步）

**适用条件**：单一数据对象、单一模块、1-3天开发周期

```
文档阶段（S0-S6）→ [上下文重置] → 实施阶段（S7-S13）
```

### 4. L_large 大项目流程（三步走）

**适用条件**：多数据对象、多模块协作、超过3天开发周期

```
整体规划（L0-L7）→ 单一模块（调用 S_simple）→ 组合测试（L19-L21）
```

## 三种方法论对比

| 方法论 | 核心理念 | 我们借鉴 |
|--------|----------|----------|
| **VibeCoding** | 跟着感觉走，能跑就行 | 对话式交互、逐步确认 |
| **GSD** | 阶段化 + 上下文重置 | 轻量级通道、原子提交 |
| **SpecCoding** | 规范先行，严格遵循 | 文档驱动、DDD 分层 |

详细对比见：[docs/01-方法论对比.md](docs/01-方法论对比.md)
