# 快速开始

> 本指南帮助你快速上手 Backend Vibe Coding 提示词库

---

## 前置准备

### 1. 选择 AI 工具

推荐使用以下支持长对话和文件操作的 AI 编程工具：

| 工具 | 优势 | 获取方式 |
|------|------|----------|
| **Claude Code** | 官方工具，上下文长，质量高 | `pip install claude-code` |
| **Cursor** | VS Code 集成，体验流畅 | https://cursor.sh |
| **OpenCode** | 国内可用，支持 question 工具 | 见 OpenCode 文档 |

### 2. 准备项目想法

明确你要开发的项目：
- 功能范围（做什么、不做什么）
- 目标用户
- 核心功能

## 方式一：新功能开发（完整流程）

适用场景：新项目、新功能、大改动

### 步骤概览

```
A_init → B1~B6 → [上下文重置] → C0 → C1~C6
  (5分钟)   (30分钟)        (0分钟)    (60分钟)
```

### 详细步骤

#### 第一步：项目初始化（A_init）

1. 复制 `prompts/A_init.txt` 内容
2. 粘贴到 AI 对话中
3. 按照提示完成项目初始化
4. 确认生成的文件

**输出**：
```
project/
├── .boundary/          # 项目边界
├── .ai-rules           # AI 编程规则
├── .aiignore           # AI 忽略文件
├── .gitignore          # Git 忽略文件
├── .env.example        # 环境变量模板
├── README.md           # 项目说明
└── scripts/            # 脚本目录
```

#### 第二步：文档生成（B1~B6）

依次复制粘贴以下提示词：

| 步骤 | 文件 | 生成内容 | 耗时 |
|------|------|----------|------|
| B1 | B1_PRD.txt | PRD.md（需求文档） | 5分钟 |
| B2 | B2_FEATURES.txt | FEATURES.md（功能梳理） | 3分钟 |
| B3 | B3_API.txt | API_DESIGN.md（接口设计） | 5分钟 |
| B4 | B4_SCHEMA.txt | SCHEMA.md（数据模型） | 5分钟 |
| B5 | B5_SERVICE.txt | SERVICE.md（服务设计） | 5分钟 |
| B6 | B6_TEST.txt | TEST_DESIGN.md（测试设计） | 5分钟 |

**注意**：B1-B6 在同一个会话中完成，保持上下文连贯。

#### 第三步：上下文重置（重要！）

B6 完成后，AI 上下文已经很长（约 50%），质量开始下降。

**操作**：
1. 关闭当前 AI 对话
2. 开启新的 AI 对话
3. 继续下一步

> **为什么需要重置？**
> - 会话越长，AI 越容易"脑补"
> - 新会话 = 干净上下文 = 稳定质量

#### 第四步：基础设施（C0）

1. 复制 `prompts/C0_Infrastructure.txt` 内容
2. 粘贴到新的 AI 对话中
3. 按照提示完成基础设施搭建

**输出**：
```
project/
├── main.py              # FastAPI 入口
├── requirements.txt     # 依赖列表
├── Dockerfile           # 容器化（可选）
├── docker-compose.yml   # 容器编排（可选）
└── src/core/
    ├── config.py        # 配置管理
    ├── database.py      # 数据库连接
    └── security.py      # 安全模块
```

#### 第五步：代码生成（C1~C6）

依次复制粘贴以下提示词（**继续在 C0 的会话中**）：

| 步骤 | 文件 | 生成内容 |
|------|------|----------|
| C1 | C1_Models.txt | 数据模型层 + 测试 |
| C2 | C2_Repository.txt | Repository 层 + 测试 |
| C3 | C3_Service.txt | Service 层 + 测试 |
| C4 | C4_API.txt | API 层 + 测试 |
| C5 | C5_Frontend.txt | Streamlit 前端验证 |
| C6 | C6_Report.txt | 测试报告 |

**注意**：C0→C1 不需要重置，因为 C0 是 C1 的直接依赖。

## 方式二：快速通道（D）

适用场景：Bug 修复、配置变更、小优化

### 步骤

1. 复制 `prompts/D_quick_fix.txt` 内容
2. 粘贴到 AI 对话中
3. 按照 5-8 步流程完成

**优势**：
- 无需走完整 13 步流程
- 从 30 分钟缩短到 5-15 分钟

## 常见问题

### Q: 什么时候需要上下文重置？

**A**: 只在 B6 → C0 之间需要重置。C0 → C1 → C2... 都在同一会话中继续。

### Q: 可以跳过某个步骤吗？

**A**: 可以灵活调整 B1-B6 的顺序，但建议完整执行 C0-C6。

### Q: 支持哪些编程语言？

**A**: 提示词基于 Python/FastAPI 设计，但可适配其他语言。

### Q: 如何查看流程图？

**A**: 见 [flowsheet.md](flowsheet.md)
