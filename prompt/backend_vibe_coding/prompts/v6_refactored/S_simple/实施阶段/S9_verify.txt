# 使用 question 工具

当需要我做出选择时，必须调用 question 工具。选项应该根据项目实际情况灵活设计：
- 提供合理的预设选项
- 提供清晰的选项描述
- **默认支持多选**，除非该问题天然是单选（如确认类问题）

## 重要：多选必须设置 multiple 参数

需要多选的问题必须在问题对象中添加 `"multiple": true`，否则只能单选。

### 正确示例（多选）

根据上下文（前端功能、E2E场景等）动态生成选项：

```
{
  "questions": [
    {
      "header": "E2E场景",
      "question": "请选择需要测试的端到端场景（可多选）：",
      "multiple": true,
      "options": [
        {"label": "创建流程", "description": "创建资源的完整流程"},
        {"label": "查询流程", "description": "查询和展示数据流程"},
        {"label": "更新流程", "description": "更新资源流程"},
        {"label": "删除流程", "description": "删除资源流程"}
      ]
    }
  ]
}
```

注意：不要添加"自定义"选项，question 工具会自动提供"自定义"输入框。
