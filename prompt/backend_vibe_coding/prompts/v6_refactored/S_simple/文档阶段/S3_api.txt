# 重要原则

1. **一步一步来**：每次只做一个任务，完成后立即暂停，等待我的输入
2. **必须提供选项**：所有需要我做出决定的问题，都必须使用 question 工具提供选项
3. **不要假设**：不要自行假设或填写任何内容

# 使用 question 工具

当需要我做出选择时，必须调用 question 工具：
- 提供合理的预设选项
- 提供清晰的选项描述
- **默认支持多选**，除非该问题天然是单选

**重要**：需要多选的问题必须添加 `"multiple": true` 参数。

# 步骤 7：生成文档

根据以上信息，生成 `docs/API_DESIGN.md`，包含以下内容：

## 文档结构

```markdown
# [项目名称] API 设计文档

**版本**: v1.0
**日期**: YYYY-MM-DD

---

## 1. API 概述

### 1.1 基础信息

- **协议**: HTTP/HTTPS
- **格式**: JSON
- **编码**: UTF-8
- **基础路径**: `/api/v1`

### 1.2 通用规范

- 认证方式: [方式]
- 请求方法: GET/POST/PUT/DELETE
- 响应格式: 统一包装 `{code, message, data}`

---

## 2. 接口列表

### 2.1 [资源名] 接口

#### 2.1.1 创建资源

**请求**:
```
POST /api/v1/[资源]
```

**请求参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| field1 | string | 是 | 字段1 |

**响应**:
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "...",
    "field1": "..."
  }
}
```

**错误码**:
| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误 |
| 409 | 资源已存在 |

#### 2.1.2 查询列表

...

#### 2.1.3 查询详情

...

#### 2.1.4 更新资源

...

#### 2.1.5 删除资源

...

---

## 3. 数据模型

### 3.1 请求模型

| 模型 | 字段 | 说明 |
|------|------|------|
| CreateXxx | field1, field2 | 创建请求 |

### 3.2 响应模型

| 模型 | 字段 | 说明 |
|------|------|------|
| XxxResponse | id, field1, created_at | 资源响应 |

---

## 4. 错误处理

### 4.1 错误码规范

| 错误码 | HTTP状态 | 说明 |
|--------|---------|------|
| 0 | 200 | 成功 |
| 1001 | 400 | 参数错误 |
| 1002 | 409 | 资源已存在 |
| 1003 | 404 | 资源不存在 |
| 1004 | 403 | 无权限 |
| 5000 | 500 | 服务器错误 |

### 4.2 错误响应格式

```json
{
  "code": 1001,
  "message": "参数错误：field1 不能为空",
  "details": {
    "field": "field1",
    "error": "required"
  }
}
```

---

## 5. API 测试计划

### 5.1 测试范围

| 测试类型 | 覆盖内容 | 优先级 |
|---------|---------|--------|
| 参数验证 | 必填、格式、范围 | P0 |
| 响应格式 | 数据结构、字段类型 | P0 |
| 错误处理 | 各种错误场景 | P0 |
| 权限控制 | 访问权限验证 | P1 |

### 5.2 测试用例

#### TC-API-001: 创建资源 - 成功

**测试目标**: 验证正常创建流程
**请求**:
```json
POST /api/v1/xxx
{
  "field1": "test"
}
```
**预期结果**: 201 Created，返回资源ID

#### TC-API-002: 创建资源 - 参数缺失

**测试目标**: 验证参数校验
**请求**: 缺少必填字段
**预期结果**: 400 Bad Request，错误码 1001

#### TC-API-003: 创建资源 - 重复创建

**测试目标**: 验证唯一约束
**请求**: 创建已存在的资源
**预期结果**: 409 Conflict，错误码 1002

#### TC-API-004: 查询详情 - 资源不存在

**测试目标**: 验证404处理
**请求**: GET /api/v1/xxx/non-existent-id
**预期结果**: 404 Not Found，错误码 1003

#### TC-API-005: 更新资源 - 权限不足

**测试目标**: 验证权限控制
**请求**: 更新无权限的资源
**预期结果**: 403 Forbidden，错误码 1004

#### TC-API-006: 列表查询 - 分页

**测试目标**: 验证分页功能
**请求**: GET /api/v1/xxx?page=1&page_size=10
**预期结果**: 返回分页数据和分页信息

### 5.3 测试覆盖率目标

| 类型 | 目标覆盖率 |
|------|-----------|
| 接口覆盖 | 100% |
| 场景覆盖 | 90% |
| 错误码覆盖 | 100% |

---

## 6. API 文档

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc
```
