# 重要原则

1. **一步一步来**：每次只做一个任务，完成后立即暂停，等待我的输入
2. **必须提供选项**：所有需要我做出决定的问题，都必须使用 opencode 的 question 工具提供选项，不能让用户手动输入
3. **不要假设**：不要自行假设或填写任何内容

# 使用 question 工具

当需要我做出选择时，必须调用 question 工具。选项应该根据项目实际情况灵活设计：
- 提供合理的预设选项
- 提供"自定义"或"其他"选项让用户输入具体内容
- 选项描述要清晰说明每个选择的特点
- **默认支持多选**，除非该问题天然是单选（如确认类问题）

## 重要：多选必须设置 multiple 参数

需要多选的问题必须在问题对象中添加 `"multiple": true`，否则只能单选。

# 步骤 8.5：完善启动脚本

更新 `scripts/dev.sh`，移除 TODO，补充完整启动命令：

```bash
#!/bin/bash
# 开发环境启动脚本

set -e

echo "Starting development server..."

# 加载环境变量
if [ ! -f .env ]; then
    echo "Warning: .env file not found, copying from .env.example"
    cp .env.example .env
fi

# 启动服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

同时创建 `scripts/dev.bat`（Windows 版本）：

```batch
@echo off
REM 开发环境启动脚本 (Windows)

if not exist .env (
    echo Warning: .env file not found, copying from .env.example
    copy .env.example .env
)

uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
