# step 1: 需求梳理审查 (PRD Final)

## Context
读取并理解以下文件：
- `<PROJECT_PATH>/docs/PRD_DRAFT.md` - 你填写的需求草稿

## Work path
<PROJECT_PATH>/docs

## Task
1. **审查**：指出需求中模糊、矛盾或无法技术实现的点。
2. **细化**：将用户故事转化为具体的"验收标准 (Acceptance Criteria, AC)"。
3. **输出**：生成正式的 `PRD.md`。

## Output Standard (`PRD.md`)
1. **结构**：
   - 1. 项目目标 (一句话)
   - 2. 功能清单 (表格：功能点 | 优先级 | 验收标准)
   - 3. 数据流向 (简易 Mermaid 图)
   - 4. 边界条件 (异常处理要求)
2. **风格**：验收标准必须是可测试的 (例如："输入错误密码 3 次锁定账户" 而不是 "安全性要好")。
3. **简洁**：去除所有背景废话，只保留开发依据。

## Action
生成 `PRD.md`。
