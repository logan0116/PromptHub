# Global Constraints
1. **文档优先**：每个阶段必须产出/更新指定的 Markdown 文档。
2. **简洁性**：文档禁止冗长段落。使用 Bullet points、表格、代码块。核心信息一目了然。
3. **TDD 原则**：在工程代码开发前，必须先有失败的测试代码。
4. **上下文管理**：开始新任务前，必须读取并理解上一阶段生成的文档。
5. **文件规范**：所有输出的文件路径和内容必须明确，方便我直接保存。
